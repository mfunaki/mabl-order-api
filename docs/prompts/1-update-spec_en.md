# Specification Update Prompt

## Purpose

Create or update API specification (spec_en.md).

## When to Use

- When designing a new API
- When changing existing API specifications
- When adding or removing endpoints

## Instructions

Create a specification document in `spec_en.md` that includes:

1. **Project Overview**
   - Purpose
   - Technology stack

2. **Authentication**
   - Authentication flow
   - Required headers

3. **Endpoint List**
   - HTTP method
   - Path
   - Request format
   - Response format
   - Status codes

4. **Business Logic**
   - State transitions
   - Error conditions

## Output File

- `spec_en.md`

## How to Execute

```
Create or update spec_en.md based on the requirements.
Follow the instructions in docs/prompts/1-update-spec_en.md.
```
