# 仕様書更新プロンプト

## 目的

API仕様書（spec.md）を作成または更新します。

## いつ使う

- 新しいAPIを設計する時
- 既存のAPIの仕様を変更する時
- エンドポイントを追加・削除する時

## 指示

以下の情報を含む仕様書を `spec.md` に作成してください：

1. **プロジェクト概要**
   - 目的
   - 技術スタック

2. **認証方式**
   - 認証フロー
   - 必要なヘッダー

3. **エンドポイント一覧**
   - HTTPメソッド
   - パス
   - リクエスト形式
   - レスポンス形式
   - ステータスコード

4. **ビジネスロジック**
   - ステート遷移
   - エラー条件

## 出力ファイル

- `spec.md`

## 実行方法

```
要件に基づいて spec.md を作成または更新してください。
docs/prompts/1-update-spec.md の指示に従ってください。
```
