mabl-order-api (1.0.0)

Download OpenAPI specification:

mablの自動テスト機能をデモンストレーションするために作成された、注文管理用のREST APIサーバーです。 意図的にステート遷移やエラーを含んでいます。

認証

ログイン（JWTトークン取得）

ユーザー認証を行い、JWTトークンを取得します。

テスト用認証情報:

  • ユーザー名: demo
  • パスワード: password

username

データ管理

データベースリセット

メモリ上のすべての注文データを削除します。

認証不要です。テスト実行前のクリーンアップに使用します。

Responses

ローカル開発サーバー

http://localhost:3000/api/reset

Response samples

  • 200
Content type
application/json
{
  • "message": "Database reset"
}

デモデータ作成

データをリセットし、デモ用の注文を1件作成します。

作成されるデータ:

  • ID: 1
  • ステータス: created

認証不要です。テスト実行前の初期データ投入に使用します。

Responses

ローカル開発サーバー

http://localhost:3000/api/seed

Response samples

  • 200
Content type
application/json
{
  • "order": {
    • "id": "1",
    • "item": "サンプル商品",
    • "status": "created",
    • "createdAt": "2019-08-24T14:15:22Z"
    }
}

支払い処理

指定された注文の支払いを処理します。

ステータス遷移:

  • created → paid

エラー条件:

  • 既に支払い済み（paid）の場合は400エラー
  • 既に発送済み（shipped）の場合は400エラー
Authorizations:
bearerAuth
path Parameters
id
required
string

発送処理

指定された注文を発送処理します。

ステータス遷移:

  • paid → shipped

エラー条件:

  • 未支払い（created）の場合は400エラー
  • 既に発送済み（shipped）の場合は400エラー
